Программа генерации случайных паролей

Синтаксис:
     ego_pass_gen [опции]

Опции:
  -minl N       Минимальная длина пароля (целое положительное число)
  -maxl N       Максимальная длина пароля (целое положительное число)
  -n N          Фиксированная длина пароля (несовместима с -minl/-maxl)
  -c N          Количество генерируемых паролей (по умолчанию 1)
  -a [символы]  Алфавит – строка символов (если без аргумента – все печатные ASCII)
  -C aADS       Набор категорий: a – строчные лат., A – заглавные, D – цифры, S – спецсимволы.
                Символы не должны повторяться.
  Опция -P задаёт вероятности. Поддерживаются два формата:

1. Список чисел по порядку:
     -P 0.2,0.5,0.3
   Для -a вероятности назначаются символам алфавита по порядку.
   Для -C вероятности назначаются категориям (a, A, D, S) по порядку.

2. Явное указание для конкретных символов или категорий:
     -P a=0.5,b=0.3,c=0.2         (для -a)
     -P a=0.7,D=0.3               (для -C)
   Ключи – отдельные символы (для -a) или буквы a/A/D/S (для -C).
   Несколько пар разделяются запятыми.

Если для какого-то элемента вероятность не указана, оставшаяся вероятность
(1 – сумма указанных) распределяется равномерно между всеми неуказанными.
Если сумма указанных вероятностей > 1, все вероятности нормализуются.
  -d символ     Добавить символ разделитель для значений опций.
  -D символ     Заменить все разделители на указанный символ.

Разделители по умолчанию: ':' и '='.
Опции могут передаваться в любом порядке. Неизвестные опции игнорируются.

Настройка проекта для сборки в студии (выполняем команды в консоле в директории проекта):
> mkdir build
> cd build
> cmake -S .. -G "Visual Studio 17 2022" -A Win32

В каталоге build создаются нужные для сборки файлы. Можно открыть проект (.sln-файл) в Visual Studio и в ней собирать. Можно открыть в Visual Studio файл CMakeLists.txt. Можно собирать проект через cmake в консоле.

Сборка проекта:
> cmake --build . -C Release
> cmake --build . -C Debug


Запуск тестов:
> cmake --build . --target test_lib
> cmake --build . --target test_src
> cmake --build . --target test_all
> ctest -C Release
> ctest -C Debug
